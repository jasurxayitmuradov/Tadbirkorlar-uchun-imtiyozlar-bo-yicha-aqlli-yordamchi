# Contributing

Thanks for your interest in contributing! Please follow these guidelines to keep
changes consistent and easy to review.

## Getting Started
1. Fork the repository and create a feature branch from `main`.
2. Install dependencies: `npm install`
3. Create a `.env.local` file (see `.env.example`).
4. Run the dev server: `npm run dev`

## Development Workflow
- Keep changes focused and small.
- Use clear commit messages.
- Update documentation when behavior changes.

## Scripts
- `npm run dev` - start Vite dev server
- `npm run build` - typecheck + build
- `npm run preview` - preview production build

## Code Style
- TypeScript strict mode is enabled.
- Use clear naming and avoid unused imports/vars.

## Reporting Issues
Provide:
- Steps to reproduce
- Expected vs actual behavior
- Logs or screenshots if applicable

## Security
If you find a vulnerability, please follow the guidance in `SECURITY.md`.
